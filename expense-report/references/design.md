# Expense Report Skill Design

> 기명카드 승인내역 → 회사 지출결의서 엑셀 자동 생성 스킬

## 1. Overview

### Problem
매월 하나카드 기명카드 사용 내역을 회사 지출결의서 양식(xlsx)에 수작업으로 옮기고, 계정과목/사용내역/동반자 등을 하나하나 분류하는 반복 작업이 발생한다.

### Solution
Claude Code 스킬 + Python 스크립트 하이브리드 구조로:
- 카드 승인내역(xls) 파싱 → 양식 자동 채움
- Notion 팀활동비 DB 매칭 → 팀 커피 항목 자동 분류
- 시간대/가맹점 기반 규칙으로 AI 자동 분류
- 수기 필요 항목만 사용자에게 제시
- 영수증 이미지 자동 첨부
- 요약 시트 캡처

### Approach
C안: 하이브리드 (Claude Code 스킬이 오케스트레이션, Python 스크립트가 데이터 처리/엑셀 생성)

## 2. Skill Interface

### 메인 스킬: `/expense {폴더경로}`

폴더경로 필수. 예시:
```
/expense /Users/ykh/Documents/drive/개인경비 지출결의서/2026/202603
```

폴더 내 필요 파일:
- `간편서비스_승인내역.xls` (필수)
- `*.jpg`, `*.jpeg`, `*.png` (선택 - 영수증 첨부용)

### 캡처 스킬: `/expense-capture {폴더경로}`

수기 입력 완료 후 요약 시트 캡처용. 예시:
```
/expense-capture /Users/ykh/Documents/drive/개인경비 지출결의서/2026/202603
```

## 4. Output: 지출결의서 Excel

### 4.1 양식 파일

원본: `/Users/ykh/Documents/drive/개인경비 지출결의서/양식_법인카드_하나 20250716_개정.xlsx`

1. `1.매출내역(원본)` — 카드 원본 데이터 (xls → 여기)

## 5. Classification Rules

우선순위순 (먼저 매칭되면 이후 규칙 스킵):

### Rule 1: Notion 팀활동비 매칭
- **조건**: Notion(사용일, 금액) == 카드(이용일, 승인금액)
- **사용내역**: "팀 커피"
- **경비 청구액**: 이용금액 전액
- **계정과목**: "복리후생비[회식비]"
- **동반자**: Notion 동반자 이름 (쉼표 구분)

### Rule 2: 카카오T택시 가승인 취소
- **조건**: 가맹점명 == "카카오T택시_가승인"
- **사용내역**: "택시 카드 선승인 취소 건"
- **경비 청구액**: 0
- **계정과목**: "여비교통비[택시]"
- **동반자**: "양경희"

### Rule 3: 카카오T일반택시(법인)
- **조건**: 가맹점명 starts with "카카오T일반택시(법인)"
- **사용내역**: "야근 택시비(배포)"
- **경비 청구액**: 이용금액 전액
- **계정과목**: "여비교통비[택시]"
- **동반자**: "양경희"

### Rule 4: 일반 택시
- **조건**: 가맹점명에 "택시" 포함 (Rule 2, 3에 해당하지 않는 것)
- **사용내역**: (수기)
- **경비 청구액**: 이용금액 전액
- **계정과목**: "여비교통비[택시]"
- **동반자**: "양경희"
- **출발지→도착지**: "본사(강남) -> 집(김포)"

### Rule 5: 카페 (Notion 미매칭)
- **조건**: 가맹점명에 카페 키워드 포함 AND Notion 매칭 안 됨
- **카페 키워드**: 스타벅스, 투썸, 바나프레소, 컴포즈, 커피, 카페, CAFE, 로스터리, 이디야, 할리스, 메가커피, 빽다방, 커핀그루나루, 드롭탑, 파스쿠찌, 엔제리너스, 탐앤탐스
- **사용내역**: "팀 커피(확인필요)"
- **경비 청구액**: 이용금액 전액
- **계정과목**: "복리후생비[회식비]"
- **동반자**: "양경희"
- **시간대 무관**: 카페 키워드가 감지되면 시간에 관계없이 적용

### Rule 6: 점심 식대
- **조건**: 승인시간 11:00~13:59 AND 비카페 AND 비PG
- **사용내역**: "점심 식대" (PG 가맹점이면 "점심 식대(확인필요)")
- **경비 청구액**: min(이용금액, 10000)
- **계정과목**: "복리후생비[식비]"
- **동반자**: "양경희"
  - 이용금액 >= 20000 (상한의 2배) → "양경희,확인필요"

### Rule 7: 저녁 식대
- **조건**: 승인시간 17:00~20:29 AND 비카페 AND 비PG
- **사용내역**: "저녁 식대" (PG 가맹점이면 "저녁 식대(확인필요)")
- **경비 청구액**: min(이용금액, 12000)
- **계정과목**: "복리후생비[식비]"
- **동반자**: "양경희"
  - 이용금액 >= 24000 (상한의 2배) → "양경희,확인필요"

### Rule 8: 수기 입력
- **조건**: 위 규칙 모두 미해당
- 모든 필드 빈칸 (셀 배경: 노란색 `FFFF00`으로 표시)
- 스킬이 사용자에게 표로 제시하여 대화형으로 입력받을 수 있음

### Rule 적용 순서와 주의사항
- Rule 1~7은 **첫 실행 시 자동 분류** 규칙. 우선순위순으로 먼저 매칭되면 이후 규칙 스킵
- Rule 5는 시간대와 무관하게 카페 키워드를 우선 감지. 점심/저녁 시간대 카페도 R6/R7이 아닌 R5로 분류됨
- Rule 6, 7에서 PG 가맹점(네이버페이 등)은 실제 가맹점을 알 수 없으므로 "(확인필요)" 태그 부여
- Rule 8: 위 어떤 규칙에도 해당하지 않는 항목은 수기 입력 대상
- **재실행 시 수기 보존**: 이전 실행에서 R8로 수기 입력한 값(계정과목, 사용내역, 경비금액, 동반자)은 재실행 시에도 보존됨

## 6. Receipt Image Attachment

### 대상 파일
폴더 내 `*.jpg`, `*.jpeg`, `*.png` 파일 (파일명 정렬순)

### 삽입 위치
"영수증 첨부" 시트, A1 셀부터 아래로 순서대로

### 이미지 크기
- **최대 너비: 4.29인치** (308.88px @72dpi)
- 원본 너비가 4.29인치 이하면 원본 크기 유지
- 초과 시 너비를 4.29인치로 축소, 높이는 비율 유지
- openpyxl 단위: 1인치 = 914400 EMU, 4.29인치 = 3,924,756 EMU

### 이미지 간 간격
- 이미지 사이 2행 간격

### 택시 영수증 검증
- 분류 결과에 택시비 항목(Rule 2, 3, 4)이 1건이라도 존재하면, 폴더 내 영수증 이미지가 반드시 있어야 함
- 이미지 파일이 없으면 **경고 메시지** 출력: "택시비 N건이 있지만 영수증 이미지가 없습니다. 영수증을 폴더에 추가한 후 다시 실행하세요."
- 엑셀은 생성하되, 영수증 첨부 시트가 비어있음을 사용자에게 알림

## 7. Summary Sheet Capture

### 방법
AppleScript로 Excel.app 제어:
1. 대상 xlsx 파일 열기
2. "요약" 시트 활성화
3. 사용 범위 선택
4. `screencapture -l {windowID}` 로 윈도우 캡처
5. 저장: 같은 폴더에 `요약.png`
6. Excel 종료 (저장 안 함)

### 실행 조건
- **수기 항목 없음** (Rule 8 해당 건 = 0): `/expense` 실행 시 자동으로 캡처까지 완료
- **수기 항목 있음**: 엑셀만 생성. 사용자에게 안내:
  > "수기 입력이 필요한 항목이 N건 있습니다. 엑셀에서 노란색 셀을 채운 후 `/expense-capture {경로}`로 요약 캡처를 실행하세요."

## 9. Skill Execution Flow

```
/expense {folder_path}
│
├─ 1. Validate folder
│   ├─ 간편서비스_승인내역.xls 존재 확인
│   ├─ 이미지 파일 목록 수집
│   └─ 폴더명에서 YYYYMM 추출
│
├─ 2. Notion query (MCP)
│   ├─ query-database-view: 팀활동비 DB 조회
│   ├─ 사용내역="커피", 사용자=양경희, 해당월 필터
│   ├─ get-users: user ID → 이름 매핑
│   └─ 결과를 JSON 파일로 저장 (/tmp/notion_data.json)
│
├─ 3. Run Python script
│   └─ python3 src/expense-report/main.py \
│        --folder {folder_path} \
│        --notion-data /tmp/notion_data.json
│   ├─ 파싱 → 매칭 → 분류 → 엑셀 생성
│   ├─ 영수증 이미지 삽입
│   └─ stdout: JSON {created_file, manual_items[]}
│
├─ 4. Present results
│   ├─ 자동 분류된 항목 요약 표
│   └─ 수기 필요 항목 상세 표
│
└─ 5. Summary capture (조건부)
    ├─ manual_items == 0 → AppleScript 캡처 실행
    └─ manual_items > 0  → "/expense-capture {path}" 안내
```

```
/expense-capture {folder_path}
│
├─ 1. 해당 폴더의 xlsx 파일 찾기
├─ 2. AppleScript: Excel 열기 → 요약 시트 캡처
└─ 3. 요약.png 저장 완료 안내
```

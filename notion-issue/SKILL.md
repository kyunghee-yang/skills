# 노션 이슈 등록

사용자가 일감 등록을 요청하면 노션 🌱 이슈 DB에 새 페이지를 생성한다.

## 입력 파싱 규칙

사용자 메시지에서 **일감명**과 **설명**을 분리한다.

**짧은 요청** (한 줄 수준):
- "로그인 버그 수정 일감 등록해줘" → 일감명: `로그인 버그 수정`, 설명: 없음
- 사용자가 말한 그대로 타이틀에 넣는다.

**긴 요청** (상세 설명 포함):
- 사용자가 배경, 조건, 기대 결과 등을 함께 설명한 경우
- 핵심을 요약해서 타이틀에 넣고, 전체 설명을 페이지 본문(content)에 작성한다.
- 타이틀은 20자 내외로 간결하게 요약한다.

## 페이지 생성

`mcp__notion__notion-create-pages` 도구를 호출한다.

### 고정 속성값

| 속성 | 값 | 비고 |
|------|-----|------|
| `이름` | 파싱한 일감명 | title 속성 |
| `상태` | `해야할 일` | 띄어쓰기 주의 |
| `담당자` | `["0f90d063-d2bb-4e09-9b39-f7bdc4883d2b"]` | 양경희 |
| `date:일정:start` | 오늘 날짜 (YYYY-MM-DD) | ISO-8601 |
| `태그` | 일감 내용에서 추론 (선택) | multi_select |

### 태그 자동 지정

일감명과 설명을 분석하여 아래 태그 중 적절한 것을 지정한다.
명확히 해당하는 태그가 없으면 비워둔다. 억지로 붙이지 않는다.

| 태그 | 해당하는 경우 |
|------|-------------|
| 조사 | 리서치, 조사, 분석 요청 |
| 기획 | 기획, 스펙 정의, 요구사항 정리 |
| 디자인 | UI/UX 디자인, 시안, 와이어프레임 |
| 검토 | 리뷰, 검토, 피드백 |
| 분석 | 데이터 분석, 원인 분석 |
| 설계 | 아키텍처, DB 설계, API 설계 |
| 구현 | 개발, 구현, 코딩, 기능 추가 |
| 테스트 | 테스트, QA 준비 |
| 디버그 | 버그 수정, 오류 해결 |
| QA | QA 진행, 품질 검증 |
| 배포 | 배포, 릴리즈 |
| 지원 | 운영 지원, CS, 고객 대응 |
| 회의 | 미팅, 회의, 싱크, 면접 |
| 인프라 | 서버, 인프라, 모니터링 |
| 데이터 | 데이터 마이그레이션, ETL |
| 보안 | 보안 점검, 취약점 |
| 문서화 | 문서 작성, API 문서 |
| 프로젝트 관리 | 일정 관리, 스프린트 |
| 교육 | 교육, 세미나, 온보딩 |
| HR | 채용, 인사, 평가 |
| 기능개선 | 기존 기능 개선, 리팩토링 |

태그는 복수 선택 가능하다. 예: 면접 → `["회의", "HR"]`

### 상위 이슈 자동 연결

일감 내용이 특정 에픽에 해당하면 `상위 이슈` relation 속성을 자동 설정한다.
페이지 생성 후 `mcp__notion__notion-update-page`로 `상위 이슈`를 업데이트한다.

| 키워드 | 상위 이슈 | 페이지 URL |
|--------|----------|-----------|
| 이력서, 채용, 면접, HR, 지원자, 코딩테스트 | 백엔드 엔지니어 채용 (RND-41957) | `https://www.notion.so/303270b243d080428018f679421dbb4b` |

설정 방법:
```json
{
  "상위 이슈": "[\"https://www.notion.so/303270b243d080428018f679421dbb4b\"]"
}
```

해당하는 에픽이 없으면 설정하지 않는다.

### 호출 예시

```json
{
  "parent": {
    "type": "data_source_id",
    "data_source_id": "84f8ed5a-b10b-400d-aa02-deb267829424"
  },
  "pages": [{
    "properties": {
      "이름": "로그인 버그 수정",
      "상태": "해야할 일",
      "담당자": "[\"0f90d063-d2bb-4e09-9b39-f7bdc4883d2b\"]",
      "date:일정:start": "2026-03-26",
      "태그": "[\"디버그\"]"
    },
    "content": ""
  }]
}
```

content에는 plain markdown만 사용한다 (heading, bullet list, bold, code block).
callout, toggle 등 Notion 전용 블록은 사용하지 않는다.

## 검증

페이지 생성 후 `mcp__notion__notion-fetch`로 생성된 페이지를 조회하여 등록 내용이 정확한지 검증한다.

### 검증 항목

조회한 속성값을 사용자의 원래 요청과 대조하여 다음을 확인한다:

| 항목 | 검증 내용 |
|------|----------|
| 이름 | 사용자가 요청한 일감명과 일치하는가 |
| 담당자 | 요청한 담당자가 모두 포함되어 있는가 |
| 일정 | 요청한 날짜/시간과 일치하는가 |
| 상태 | `해야할 일`로 설정되어 있는가 |
| 태그 | 일감 내용에 맞는 태그가 지정되었는가 (없으면 비어있는가) |
| 본문 | 긴 요청의 경우 설명이 content에 작성되었는가 |

### 검증 결과 처리

**모두 일치하면** 정상 등록으로 판단하고 응답을 출력한다.

**불일치 항목이 있으면** 어떤 값이 다른지 사용자에게 알려주고, `mcp__notion__notion-update-page`로 수정할지 확인한다.

## 응답

검증 통과 시 다음을 출력한다:
- 이슈 ID (RND-XXXX) — `userDefined:ID` 속성에서 추출
- 생성된 페이지 URL
- 등록한 일감명, 상태, 담당자, 일정 요약
- 검증 결과

**출력 예시:**
```
RND-1234 | 로그인 버그 수정
https://www.notion.so/xxxxx
상태: 해야할 일 | 담당자: 양경희 | 일정: 2026-03-26
검증: 이름 ✅ 담당자 ✅ 일정 ✅ 상태 ✅ 태그 ✅
```
